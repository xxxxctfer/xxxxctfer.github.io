反射型XSS

利用

这个算是最开始接触到的XSS了，也是最简单的XSS吧，直接将JS代码插入到HTML中，当别人访问的时候就会自动触发，然后造成危害，举个例子。

1
2
3

<?php
   echo $_GET["uname"];
?>

当我们访问这个PHP代码对应的界面的时候 /?uname=时候，就会触发反射型XSS。 可以看到alert是成功触发了，说明正常执行了我们写入的js代码，所以我们就可以写一些别的恶意JS代码进去，盗取cookie之类的了。

修复

这里需要根据情况来判定，先说明我们这里的情况，是输入到了HTML当中，所以我们需要用HTML转义的函数去进行操作，即htmlspecialchars()，如果是输入到JS当中的话，我们就需要换一个函数，即json_encode

1
2
3
4
5
6
7
8

<?php
if (isset($_GET["uname"])) {
    $safe_uname = htmlspecialchars($_GET["uname"], ENT_QUOTES, 'UTF-8');
    echo $safe_uname;
} else {
    echo "请传入 uname 参数（例如：?uname=你的名字）";
}
?>

可以看到，修改之后是顺利转义掉了。还有对应的JS代码的情况，这里就给代码出来，不实现看了。

存储型XSS

利用

说实话这个一开始学着有点迷迷糊糊的，没太搞懂什么意思，这次正好整一下。先说原理吧，就是从数据库中提取信息出来，并显示到页面的过程中，没有对数据库内的信息进行处理，可能导致存储型的XSS漏洞出现。下面是一个测试代码。

<?php
// 连接数据库（请替换为你的数据库账号密码）
$conn = mysqli_connect('localhost', 'root', '123456', 'xss_demo');
mysqli_set_charset($conn, 'utf8mb4');

// 1. 处理用户提交的留言（存储到数据库）
if (isset($_POST['content'])) {
    $content = $_POST['content']; // 直接接收用户输入，无任何过滤！
    $sql = "INSERT INTO messages (content) VALUES ('$content')";
    mysqli_query($conn, $sql);
}

// 2. 读取所有留言并展示（从数据库读取后直接输出）
$sql = "SELECT content FROM messages ORDER BY id DESC";
$result = mysqli_query($conn, $sql);
?>

<!DOCTYPE html>
<html>
<head>
    <meta charset="utf-8">
    <title>有存储型XSS的留言板</title>
</head>
<body>
    <h3>留言板（有存储型XSS漏洞）</h3>
    <!-- 留言提交表单 -->
    <form method="POST">
        <textarea name="content" rows="3" cols="50" placeholder="请输入留言"></textarea>
        <br>
        <button type="submit">提交留言</button>
    </form>

    <hr>
    <h4>历史留言：</h4>
    <?php
    // 危险：从数据库读取的内容直接输出，无转义！
    while ($row = mysqli_fetch_assoc($result)) {
        echo "<div>{$row['content']}</div><br>";
    }
    ?>
</body>
</html>

可以看到留言板功能，最好的提取输出的时候没有做转义处理，导致了出现XSS漏洞，并且是存在于数据库当中，在页面当中不会显示出来。我们新建一个数据库看看吧。

DOM型XSS

利用

这个也是一开始不是很理解的XSS，简单看看吧。简单来说就是代码将我们的输入写进了DOM里面去并且没有转义，从而导致了触发这个漏洞，看一下下面这个测试代码吧。

<!DOCTYPE html>
<html lang="zh-CN">
<head>
    <meta charset="UTF-8">
    <title>有DOM型XSS的欢迎页</title>
</head>
<body>
    <h1>欢迎访问！</h1>
    <div id="welcomeMsg">请在URL后添加 #用户名 查看欢迎语</div>

    <script>
        // 1. 从URL的hash（#后面的内容）获取用户输入
        // 比如访问：http://localhost/dom-xss.html#张三
        // 则 hash = "#张三"，截取后得到 "张三"
        const hash = window.location.hash;
        const username = hash.substring(1); // 去掉#号

        // 2. 漏洞核心：直接将用户输入插入到DOM中（无任何转义！）
        if (username) {
            // 方式1：innerHTML（最常见的漏洞点）
            document.getElementById('welcomeMsg').innerHTML = `欢迎你，${username}！`;
            
            // 补充：以下写法也有漏洞（本质都是插入未转义的HTML）
            // document.write(`欢迎你，${username}！`);
            // document.body.innerHTML += `欢迎你，${username}！`;
        }
    </script>
</body>
</html>

可以看到这里是使用的innerHTML函数去触发的，当然下面的document.write和document.body.innerHTML也会触发，具体的去查一下函数逻辑即可。我们去测试一下。 可惜被浏览器的安全机制waf掉了，这里暂时就执行不了了，但是我们可以去看写入的位置是哪里。 可以看到，如果没有被waf掉，这里是正常输入就会触发XSS了。
