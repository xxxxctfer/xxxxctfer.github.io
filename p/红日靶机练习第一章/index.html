内网信息收集

我们先来做一下内网的信息收集。分下面几个步骤进行吧，如果后续有新东西也会继续补充。

  1. 确认木马的权限 getpid和getsystem，可以确认当前的权限
  2. 查看局域网内的其他主机名 常用命令是net view，其他的也放在下面了。
  3. 确认域后缀，是哪个域的，有几个域，域主机名 shell ipconfig/all，shell net view /domain，run net group "domain controllers" /domain。
  4. 找到域控服务器的主机和内网IP 加上域后缀来ping对应的主机，就可以找到对应的域控服务器了。shell ping 主机名.域后缀

知识点解析

黄金票据

打了这么多命令，但是其实不是很清楚黄金票据是什么东西，简单来说就是一个永久的身份信息，当年控制了别的内网主机，或者说之前那个断掉了，你用一台新的内网主机，还是可以通过这个票据，登录成域控

1

mimikatz kerberos::ptt 0-40e00000-Adminstrator@krbtgt~god.org-god.org.kirbi

哈希码横向移动

前面我们是使用了一个方式直接去抓到明文密码的，但是原理没有介绍，这里介绍一下。 实际上就是运用了哈希码横向移动，是什么意思呢。其实是一个残留。简单来说就是域控登录过我们当前的这台机子，有一个残留的登录凭证，通过这个登录凭证，我们可以直接登录到域控里面，上传木马，然后拿到域控服务器，进而控制的整个域。下面是一个流程图。

这就是CS的获取密码到上线后门的一个操作过程。

inject操作

还记得我们前面做权限维持的时候，我们通过inject操作去对一个exe文件注入了木马，但是具体操作是怎么样的，我们探究一下。 具体流程其实是开辟内存去注入木马进去的，我们来看看流程。

1
2
3
4
5

+-------------------+
| 正常代码 (exe)    |
| 正常数据          |
| 正常堆/栈         |
+-------------------+

这个是正常情况下的exe进程。我们使用OpenProcess 打开目标进程，获得读写权限，然后使用VirtualAllocEx 在目标进程开辟内存。

1
2
3
4
5
6
7

LPVOID pRemoteMem = VirtualAllocEx(
    hProcess,      // 目标进程
    NULL,          // 系统自动分配地址
    大小,          // ShellCode 长度
    MEM_COMMIT | MEM_RESERVE,  // 提交内存
    PAGE_EXECUTE_READWRITE      // 权限：可读、可写、可执行
);

然后我们的内存空间就会变化为下面这样。

| 【新开辟的内存页】 ← 这是给木马准备的
| 权限：RWX          ← 关键！可写可执行
+-------------------+

    NULL
