<?php
// 存在文件包含漏洞的代码
// 场景：模拟网站通过参数加载不同页面
$page = $_GET['page']; // 直接接收用户输入，无任何过滤

// 直接将用户输入作为包含路径，触发漏洞
if (isset($page)) {
    include $page; // 核心问题：未过滤、未校验的用户输入传入include
}
?>

直接包含一下我们的flag文件。 很明显就直接包含进去了，这里是包含的txt文件，所以直接显示到了页面上，如果包含的是一个脚本文件，就会直接执行，或者被连接利用。

修复

修复方案：比如白名单过滤，只能去包含白名单中有的文件，可以有效防止任意文件包含。

<?php
// 安全版本：白名单过滤 + 输入校验
// 1. 接收用户输入（仍保留，但后续会校验）
$page = $_GET['page'];

// 2. 定义合法文件白名单（核心修改点1）
// 只允许包含这些文件，新增/删除文件只需修改此列表
$allowed_pages = ['home.php', 'about.php', 'contact.php'];

// 3. 校验输入（核心修改点2）
// 只有输入在白名单内，才执行包含；否则加载默认首页
if (isset($page) && in_array($page, $allowed_pages)) {
    include $page;
} else {
    include "home.php"; // 非法输入默认加载安全页面
}
?>

这里就会直接waf掉我们一开始的payload。我们还可以从配置上面入手，比如将open_basedir修改为一个路径，然后限制掉能够包含的路径，静止别的路径，以及在代码中限制。

<?php
// 安全版本：路径固化 + 文件名白名单
// 1. 接收用户输入（仅允许传入文件名前缀，如 home、about）
$page = $_GET['page'];

// 2. 定义合法文件名前缀白名单
$allowed_prefixes = ['home', 'about', 'contact'];

// 3. 固化包含路径（核心修改点：路径固定为 ./pages/）
$base_path = './pages/';

// 4. 校验输入并拼接路径
if (isset($page) && in_array($page, $allowed_prefixes)) {
    // 拼接固定路径 + 合法前缀 + 固定后缀 .php
    include $base_path . $page . '.php';
} else {
    // 非法输入加载默认文件
    include $base_path . 'home.php';
}
?>

这个也是OK的，限制了路径。当然还有一些字符限制，协议限制等等，可以控制很多情况下的文件包含。

<?php
$page = $_GET['page'];
$allowed_pages = ['home.php', 'about.php', 'contact.php'];

// 补充修改：过滤输入中的路径遍历字符
if (isset($page)) {
    // 替换 ../、\ 等危险字符
    $page = str_replace(['../', '..\\', '/', '\\'], '', $page);
}

// 仍需白名单校验（核心）
if (isset($page) && in_array($page, $allowed_pages)) {
    include $page;
} else {
    include "home.php";
}
?>

这些都可以有效的限制一些文件包含漏洞。

危害

  1. 包含日志文件获取webshell
  2. 利用伪协议获取信息和获得敏感信息等操作。
  3. 图⽚上传并包含图⽚shell
  4. php_ zip命令压缩协议获取webshell，获取压缩包中的木马文件
  5. phar://协议获取木马文件
