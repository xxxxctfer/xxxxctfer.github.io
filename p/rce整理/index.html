RCE整理

回顾一下TOP10，整理一下

2026-03-06

说实在话的，RCE没有什么特别好总结的，简单讲讲危害算了。

危害

  1. webshell权限，获取服务器的控制权
  2. 任意读写文件，帮助其他漏洞的利用
  3. 权限提升和权限维持，以及持续后门

基本上来说，RCE漏洞就意味着服务器的权限已经拿到了，很少有人会在RCE漏洞的后面加过滤了，如果非要执行命令，那会在一开始的时候加过滤，绕过之后写文件进去就行。

<?php
// 漏洞点：直接将用户GET参数传入eval执行，无任何过滤
$code = $_GET['cmd'];
if (!empty($code)) {
    eval("echo '执行结果：'; " . $code . ";"); // $code可控，触发RCE
}
?>

代码层面的rce漏洞大概跟这个差不多，主要就是eval函数的调用之类的，导致可以控制写入命令执行的函数，然后直接执行。

修复

在不影响功能的前提下修复代码，基本上就是下面这几种方法。

  1. 采用白名单机制（仅允许执行指定的代码 / 命令），拒绝黑名单（黑名单易被绕过）
  2. 过滤特殊字符，禁止命令 / 代码拼接
  3. 最小权限原则（运行 PHP 的服务器用户仅拥有必要权限，避免 root / 管理员权限） 如果只执行PHP函数的话，可以用这种白名单修复。

<?php
$code = $_GET['cmd'];
if (!empty($code)) {
    // 1. 仅允许指定的安全操作（白名单）
    $allowed_operations = ['phpinfo']; // 仅允许执行phpinfo
    if (in_array($code, $allowed_operations)) {
        // 2. 避免直接拼接代码，用函数调用方式
        if ($code == 'phpinfo') {
            phpinfo();
        }
    } else {
        die("错误：仅允许执行 " . implode(',', $allowed_operations) . " 操作");
    }
}
?>

<?php
$cmd = $_GET['command'];
if (!empty($cmd)) {
    // 1. 命令白名单：仅允许指定的安全命令
    $allowed_cmds = ['ipconfig', 'dir', 'ls', 'whoami'];
    // 2. 过滤特殊字符：禁止拼接额外命令
    $cmd = preg_replace('/[;|&<>$\`]/', '', $cmd);
    // 3. 校验命令是否在白名单内
    if (in_array($cmd, $allowed_cmds)) {
        echo "<pre>";
        system($cmd);
        echo "</pre>";
    } else {
        die("错误：仅允许执行 " . implode(',', $allowed_cmds) . " 命令");
    }
}
?>
