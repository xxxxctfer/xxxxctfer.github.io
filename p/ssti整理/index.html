SSTI整理

回顾一下TOP10，整理一下

2026-03-06

总结一下SSTI漏洞，从全名理解，是模板注入漏洞，两点点吧，一个是模板，一个是注入，也就是说这个漏洞是在一个模板的解析之下，由于恶意输入导致解析错误，形成注入的一种漏洞。

利用

写个测试代码吧，这个漏洞在PHP，python，Java中都有，这里写经典的python的。

from flask import Flask, request

app = Flask(__name__)

@app.route('/')
def index():
    # 漏洞核心：直接将用户GET参数传入模板渲染函数，无任何过滤
    # name参数可控，攻击者可构造恶意模板语法
    name = request.args.get('name', 'Guest')
    # render_template_string会解析模板语法，导致注入
    html = f"Hello, {name}!"  # 等价于 render_template_string("Hello, {{name}}!", name=name)
    return app.jinja_env.from_string(html).render()

if __name__ == '__main__':
    # 运行服务：监听所有地址，端口5000，调试模式（方便复现）
    app.run(host='0.0.0.0', port=5000, debug=True)

很明显的一个存在漏洞的情况啊，然后我们可以打一下rce的链子看看。 很简单一个打法啊，config直接获取，然后往下面打的。

危害

SSTI的危害主要看目的是什么，大致可以是以下几种吧。

  1. 进行命令执行，拿下服务器权限
  2. 任意文件读取，读取敏感信息
  3. 打内存马，也是拿服务器权限 应该主要是这些。

from flask import Flask, request, render_template_string  # 导入规范的渲染函数

app = Flask(__name__)

@app.route('/')
def index():
    # 获取用户输入（无需手动escape，Jinja2会自动转义）
    name = request.args.get('name', 'Guest')
    # 核心修复：将用户输入作为变量传入，而非拼接模板字符串
    # "Hello, {{name}}!" 是固定模板，name是纯文本变量，不会被解析为模板语法
    return render_template_string("Hello, {{name}}!", name=name)

if __name__ == '__main__':
    app.run(host='0.0.0.0', port=5000, debug=False)

这个修复还是有挺多的，这个应该算比较彻底的一种了，直接变成字符串输出。还可以加上一些黑白名单，这里就没有加了，基本上就是一些关键词，还有{{，}}之类的过滤。
