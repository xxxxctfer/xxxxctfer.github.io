<?php
// 获取用户输入的 XML（例如来自 POST 请求）
$xml = file_get_contents('php://input');

// 创建 DOMDocument 对象
$dom = new DOMDocument();
// 启用外部实体处理（漏洞点）
$dom->substituteEntities = true;      // 允许替换实体
$dom->loadXML($xml, LIBXML_NOENT);    // LIBXML_NOENT 会将实体替换为内容

// 输出解析后的内容
echo $dom->saveXML();
?>

这里就直接进行了任意文件读取。

危害

1、文件读取：定义一个包含文件内容的外部实体，然后在应用的响应里返回它。 2、SSRF攻击：把外部实体定义成一个指向后端系统的URL，让服务器去访问它 3、带外数据泄露：把服务器上的敏感数据偷偷地传到黑客控制的系统上。 4、错误消息泄露：触发一个包含敏感数据的XML解析错误消息。

修复

  1. 最直接的修复：禁用外部实体加载器。在解析 XML 之前调用 libxml_disable_entity_loader(true)。这个函数会全局禁止所有 libxml 操作加载外部实体。
  2. 针对 PHP 8.0+ 的推荐修复：使用 libxml_set_external_entity_loader()
  3. 使用安全的解析选项：在 loadXML() 或 load() 时指定合适的选项，可以避免加载外部实体，但需要结合 libxml_disable_entity_loader 或实体加载器才能彻底阻断
  4. 完全禁用 DTD 解析
  5. 一些其他方式： 输入验证：如果可能，对 XML 结构进行白名单验证，拒绝包含 或外部实体的请求。 最小权限原则：确保 Web 服务器运行账户对文件系统只有最小必要权限，防止读取敏感文件。 使用替代格式：如果 XML 不是必须的，可考虑使用 JSON 等不易受到实体攻击的数据格式。
