WEB机

我们进行一下信息收集，只知道IP是192.168.200.80 可以看到有服务，但是没什么信息，还是先进行信息收集吧。

信息收集

当正常信息收集来做，先探测存活主机。

1

arp-scan -l

发现两个存活主机，进行端口探测。

1
2
3

nmap -p- IP
# 可以得知系统版本和漏洞信息
nmap -A -v IP

开的很多啊，我们简单分析一下，80端口和7001端口是web服务，7001是weblogic的服务，然后445是smb的端口，1433是mysql的端口，3389是远程控制的端口。我们主要还是从web服务打进去，别的估计都还有防火墙保着的。 看一下这个201的主机，没开web服务，前面说的一些都有，问题不大，我们先打这个80的主机。

web服务

前面知道开了80端口和7001端口，直接去访问啥都没有，我们来扫一下目录。 OK，80端口没扫到什么东西，7001端口倒是东西挺多的，我们直接去看这边的东西。 很简单可以看出是weblogic的一个框架(其实7001端口可以看出来，这里是多看了一步再下判断) 可以看到有版本，我们直接用工具看有没有漏洞可以直接利用。 工具链接：https://github.com/KimJun1010/WeblogicTool 这个检查一个漏洞，有了就直接停了，但是问题不大，反正可以用就行。然后这里可以选择shell上次或者内存马注入，shell上传的话，需要一个jsp木马，我们来试试内存马注入能不能用。 可以访问看下有没有注入进去，可以访问我们就可以直接进行连接。 成功连上哥斯拉，拿到内存马。

内网攻击

OK，拿到服务器的权限之后我们就要往内网打了。

CS上线

拿到内存马之后我们就可以准备上线CS了。因为我们这里权限够，可以直接去关闭防火墙，方便后面上线。

1
2
3

netsh advfirewall show all state  		  //查看防火墙状态
netsh firewall set opmode disable             #winsows server 2003 之前
netsh advfirewall set allprofiles state off   #winsows server 2003 之后

但是这里好像是已经关掉了，而且哥斯拉这里会乱码，因为回显是gbk编码，而哥斯拉默认UTF-8编码接收，所以直接乱码了，我们去虚拟机里面执行看看。 OK啊，全开的，我们弄一个镜像留着，到时候回来试试，开着防火墙可不可以上CS。 OK，防火墙关掉了，我们上线CS。 简单上线CS。

权限维持

依旧权限维持，简单选择一下。 选一个顺眼的。这里还是选了2564 2024 explorer.exe x64 1 DE1AY\de1ay这个，直接inject就行。 注入了一下，结果发现权限不够，难绷。

OK，我们再注入一下。(难绷，这里注入进来之后权限掉了，刚刚没注意看，这个是普通用户，重新来了一遍，这里最好用spawn x64，要用inject也要去用高权限的cmd文件) 更新图片： 原图片： 简单处理一下后续。(这里kill错值了，应该kill 1780的，问题不大，后面还是kill了)
