成功注入进去了，可以看到报错回显出来的数据库名，成功进行了SQL注入。

危害

很明显可以看到SQL注入的危害啊，但是还是稍微总结一下吧。

  1. 数据泄露：通过SQL注入，你可以获取数据库中所有的数据，可以进行脱库
  2. 权限提升：通过一个在数据库中查询到权限，到控制整个数据库的权限，再到提升到这个服务器的权限
  3. 写文件：可以通过某些函数直接往服务器里面写webshell或者病毒进去，对服务器造成危险

<?php
// 1. 连接数据库（生产环境建议封装成函数/类）
$conn = mysqli_connect("localhost", "db_user", "db_pass", "test_db");
if (!$conn) {
    die("数据库连接失败"); // 生产环境改为日志记录，不暴露错误
}

// 2. 获取并过滤用户输入（基础清洗）
$username = trim($_POST['username'] ?? '');
$password = trim($_POST['password'] ?? '');

// 3. 准备预处理语句（? 是参数占位符，无任何拼接）
$stmt = mysqli_prepare($conn, "SELECT * FROM users WHERE username = ? AND password = ?");
// 4. 绑定参数（ss 表示两个参数都是字符串类型，i=int, d=double）
mysqli_stmt_bind_param($stmt, "ss", $username, $password);
// 5. 执行语句
mysqli_stmt_execute($stmt);
// 6. 获取结果
$result = mysqli_stmt_get_result($stmt);

// 7. 业务逻辑（示例：登录验证）
if (mysqli_num_rows($result) > 0) {
    echo "登录成功";
} else {
    echo "用户名/密码错误";
}

// 8. 关闭资源（生产环境建议自动回收，或用try-finally）
mysqli_stmt_close($stmt);
mysqli_close($conn);
?>

// 错误示例：直接拼接表名（存在注入风险）
// $table = $_GET['table'];
// $sql = "SELECT * FROM $table";

// 正确示例：白名单过滤
$allowed_tables = ['users', 'orders', 'products']; // 仅允许这些表
$table = $_GET['table'] ?? 'users';
if (!in_array($table, $allowed_tables)) {
    $table = 'users'; // 非法输入则默认值
}
$sql = "SELECT * FROM `$table`"; // 反引号避免关键字冲突

部署 Web 应用防火墙（WAF）：

  • 开源方案：ModSecurity（Apache/Nginx 模块）、Naxsi（Nginx）；
  • 商业方案：云厂商 WAF（阿里云、腾讯云等）；
  • 作用：拦截包含 SQL 注入特征的请求（如 UNION、SELECT *、-- 等），作为最后一道防线。 定期安全审计：
  • 使用工具扫描：SQLMap（本地测试）、AWVS、Nessus；
  • 定期审查代码：重点检查拼接 SQL 的地方；
  • 定期更新数据库 / PHP 版本：修复已知的安全漏洞。
