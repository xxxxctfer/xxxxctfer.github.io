<?php
// 定义一个存在危险魔术方法的类
class TestClass {
    public $data;

    // 析构方法：对象销毁时自动执行
    public function __destruct() {
        // 危险操作：直接执行用户可控的$data变量（核心漏洞点）
        eval($this->data);
    }
}
// 模拟用户输入（攻击者可控制这个参数）
$user_input = $_GET['payload'];
// 反序列化用户输入的字符串（无任何过滤，触发漏洞）
if (!empty($user_input)) {
    unserialize($user_input);
}
?>

主要看看效果，实际上没有打pop链。

打了一个phpinfo()出来，实际上这里的操作是要看代码当中给了什么东西，才能去利用什么东西，绝大部分的情况下，都是需要结合其他漏洞的利用方式的，比如这里就是代码执行出来的。

也可以打命令执行，主要看给的是什么。

危害

这个漏洞的危害比较多元化，主要要看给了什么危险类，这里简单说一些吧。

  1. 结合RCE漏洞，有的代码里面会使用eval进行处理，设置不好就会触发RCE漏洞
  2. SQL注入漏洞，有的环境下是用反序列化去进行SQL查询，可能存在SQL注入漏洞
  3. SSRF漏洞，这个算比较容易见到的了，有个原生类的__call方法就可以打
  4. 文件上传漏洞，这种不常见，但是也会有吧，貌似就是TOP10，包括一些漏洞都可以有

修复

在不修改这个功能的前提下，要做出修复，只能去修改掉代码里面的一些危险操作，比如限制可以被控制的方法之类的。

<?php
class TestClass {
    public $data;

    public function __destruct() {
        // 1. 过滤危险函数，禁止执行eval、system等
        $unsafe_functions = ['eval', 'system', 'exec', 'passthru', 'file_put_contents'];
        foreach ($unsafe_functions as $func) {
            if (strpos($this->data, $func) !== false) {
                die("非法操作：禁止执行危险函数");
            }
        }
        // 2. 仅允许安全的操作（示例：仅输出文本）
        echo "合法数据：" . htmlspecialchars($this->data);
    }
}

$user_input = $_GET['payload'];
if (!empty($user_input)) {
    // 3. 反序列化前校验：仅允许TestClass类，且数据格式合法
    $serialized_data = $user_input;
    // 校验类名（防止注入其他恶意类）
    if (preg_match('/^O:9:"TestClass":\d+:{.*}$/', $serialized_data)) {
        $obj = unserialize($serialized_data);
        // 额外校验：确保$data是字符串且长度合理
        if (!is_string($obj->data) || strlen($obj->data) > 100) {
            die("数据格式非法");
        }
    } else {
        die("仅允许TestClass类的序列化数据");
    }
}
?>

可以看到，在方法里面限制了危险函数的调用，可以防止一定的RCE调用。然后还有一些类名的校验之类的。大体思路就是限制用户的可操作，可修改范围，增加利用难度和存在漏洞的可能性。在PHP反序列化中还是比较好用的。

这里也同时提出一个Java反序列化不同的地方吧，因为Java他环境的特殊性，往往一个站点会放很多jar包进去，很难通过黑名单的方式限制，限制了也可以找到绕过的手法，所以这个思路在Java反序列化中限制作用没有在PHP中大。

__construct()  #每次创建新对象时先调用此方法
__destruct()  #某个对象的所有引用都被删除或者销毁时调用（没有变量指到当前对象时也会被触发，如 a:2:{i:0;O:4:"User":0:{}i:0;s:3:"xxx";}，被覆盖后没有变量指向User对象）
__toString()  #把类被当做一个字符串使用时调用
__wakeup()  #使用unserialize函数，反序列化恢复对象之前时调用
__sleep()  #使用serialize()函数，序列化对象之前时调用
__call()  #在对象中，调用不存在的方法或调用权限不足时调用
__callstatic()    #在静态上下文中，调用不可访问的方法时触发
__get()  #访问不存在的成员变量时调用
__set()   #设置不存在的成员变量时调用
__invoke()  #当尝试以调用函数的方式调用一个对象时触发
__autoload()  #尝试加载未定义的类
__isset()   #在不可访问的属性上调用isset()或empty()触发
__unset()   #在不可访问的属性上使用unset()时触发
