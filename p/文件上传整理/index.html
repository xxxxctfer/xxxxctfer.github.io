文件上传整理

回顾一下TOP10，整理一下

2026-03-04

原理

文件上传漏洞是一个最简单常见的漏洞，一般来说，网站存在文件上传功能点，就可以根据这个功能点以及对应服务器的框架，语言情况上传对应的恶意代码，经过服务器的解析之后在服务器中成功上传webshell

思路过程

  1. 测试是否为正常的文件上传功能点
  2. fuzz测试可上传的文件类型(.h和.u)
  3. fuzz测试所有存在的过滤(单字符，组合字符串，函数等)
  4. 尝试绕过过滤进行上传

危害

这个漏洞的危害很明显，可以上传webshell，也可以根据当前服务器的其他漏洞进行利用，最常见的危害就是控制服务器，进而控制主机，而后进行内网穿透等操作。

防护

要防御这个漏洞可以根据以下两个角度入手，以及一个安全修复方案。

1. 代码角度
   1.1 采用白名单策略，严格限制上传文件的后缀名。
   1.2 上传文件重命名，尽量少的从客户端获取信息。
   1.3 尽量避免 Web 用户修改上传白名单。
   1.4 文件内容检测。（不是图片数据不收）
   1.5 进行二次渲染，过滤掉图片马中的恶意代码。
   1.6 避免文件包含漏洞。
   1.7 严格处理文件路径，防御 00 截断 漏洞。
   1.8 避开 、 . 、 ::$DATA 等 windows 特性。
2. 服务器角度
   2.1 及时更新 Web 容器，防止解析漏洞产生。
   2.2 严格控制权限， rw- 或者 r-x 。
   2.3 建立单独的文件存储服务器，类似于站库分离。
3. 安全修复方案
   3.1 后端验证：把处理逻辑放到后端，增加攻击者的试错时间。
   3.2 后缀检测：指定黑名单、白名单机制。
   3.3 MIME 检测：基于上传类型检测。
   3.4 内容检测：对文件头、文件内容完整性检测。
   3.5 自带函数过滤：参考 uploadlabs 函数。

// 检查是否有文件上传
if ($_FILES['file']['error'] === UPLOAD_ERR_OK) {
    // 漏洞点1：直接使用客户端提交的文件名，未做任何过滤
    $filename = $_FILES['file']['name'];
    // 漏洞点2：固定上传目录，且目录可访问（如www根目录下的upload文件夹）

    } else {
        echo "文件上传失败！";
    }

<?php
header("Content-Type: text/html; charset=utf-8");

// 允许上传的文件类型（白名单）
$allowed_types = ['image/jpeg', 'image/png', 'image/gif'];
// 允许上传的文件后缀（白名单）
$allowed_extensions = ['jpg', 'jpeg', 'png', 'gif'];
// 上传目录（建议放在Web根目录外，若必须在根目录，需设置.htaccess禁止执行脚本）
$upload_dir = './upload/';

// 检查目录是否存在，不存在则创建
if (!is_dir($upload_dir)) {
    mkdir($upload_dir, 0755, true);
}

// 检查文件上传是否成功
if ($_FILES['file']['error'] === UPLOAD_ERR_OK) {
    // 1. 获取文件的真实MIME类型（通过finfo，而非客户端提交的type）
    $finfo = new finfo(FILEINFO_MIME_TYPE);
    $file_mime = $finfo->file($_FILES['file']['tmp_name']);
    
    // 2. 获取文件后缀（转换为小写，避免大小写绕过）
    $file_ext = strtolower(pathinfo($_FILES['file']['name'], PATHINFO_EXTENSION));
    
    // 3. 校验MIME类型和后缀
    if (!in_array($file_mime, $allowed_types) || !in_array($file_ext, $allowed_extensions)) {
        die("错误：仅允许上传jpg/jpeg/png/gif格式的图片文件！");
    }
    
    // 4. 生成随机文件名（避免覆盖或文件名注入）
    $random_name = uniqid() . '.' . $file_ext;
    $upload_path = $upload_dir . $random_name;
    
    // 5. 移动文件并检查是否成功
    if (move_uploaded_file($_FILES['file']['tmp_name'], $upload_path)) {
        echo "文件上传成功！路径：" . $upload_path;
    } else {
        die("文件上传失败！");
    }
} else {
    die("请选择要上传的文件，或检查文件上传是否出错！");
}
?>
